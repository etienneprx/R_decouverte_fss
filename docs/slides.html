Exemple 1 : Données quantitatives

Situation : Vous avez un sondage de 5,000 répondants avec 50 variables.

Objectif : Comparer le score moyen de satisfaction selon la région et le groupe d’âge.

Dans Excel :

  • Trier manuellement les colonnes
  • Créer des formules pour chaque calcul
  • Copier-coller pour chaque sous-groupe
  • Refaire tout si les données changent
  • Risque d’erreurs à chaque étape

Avec R :

# En 3 lignes, on obtient :
donnees %>%
  group_by(region, age_groupe) %>%
  summarise(moyenne = mean(score),
            ecart_type = sd(score),
            n = n())
  • Résultats instantanés pour tous les groupes
  • Reproductible : relancer = mêmes résultats
  • Données mises à jour? On relance le code.

Exemple 1 : La puissance du code

Le même code s’adapte à n’importe quel jeu de données

# Aujourd'hui : sondage électoral
sondage %>%
  group_by(region) %>%
  summarise(
    intention_PLC = mean(vote_lib),
    intention_PCC = mean(vote_cons)
  )
# Demain : données de santé
patients %>%
  group_by(groupe_age) %>%
  summarise(
    tension_moyenne = mean(tension),
    n_patients = n()
  )


Le principe clé

Vous apprenez la logique une fois, puis vous l’appliquez à tous vos projets. Pas besoin de tout réapprendre à chaque nouvelle analyse.

Exemple 2 : Données qualitatives

Un préjugé courant : “R, c’est juste pour les stats et les chiffres.”

La réalité : R permet aussi d’analyser des données textuelles issues d’entretiens, de documents ou de réponses ouvertes.

Précision importante

On parle ici d’analyser des données qualitatives avec R (fouille de texte, comptage, visualisation) — ce qui est différent de l’analyse qualitative au sens méthodologique (interprétation, théorisation ancrée, etc.). R est un outil qui peut soutenir votre démarche, pas la remplacer.

Exemple 2 : Données qualitatives

Situation : Vous avez 30 entretiens semi-dirigés à analyser.

Objectif : Identifier les thèmes récurrents et comparer leur fréquence entre participants.

Approche traditionnelle :

  • Lire chaque entretien plusieurs fois
  • Surligner manuellement les passages
  • Compter les occurrences à la main
  • Difficile de repérer des patterns subtils
  • Très chronophage

Avec R :

# Trouver tous les passages sur un thème
entretiens %>%
  filter(str_detect(texte,
    "stress|anxiété|pression"))

# Compter les thèmes par participant
entretiens %>%
  group_by(participant) %>%
  summarise(
    nb_stress = sum(str_detect(texte, "stress")),
    nb_positif = sum(str_detect(texte, "satisf"))
  )

Exemple 2 : Analyse de contenu avec R

R peut aller beaucoup plus loin dans l’analyse de données textuelles

# Exemple : Extraire les mots les plus fréquents par thème
entretiens %>%
  unnest_tokens(mot, texte) %>%           # Séparer chaque mot
  anti_join(mots_vides_fr) %>%            # Retirer "le", "la", "de"...
  count(theme, mot, sort = TRUE) %>%      # Compter par thème
  group_by(theme) %>%
  top_n(10)                               # Garder le top 10

Ce que R permet avec les données textuelles

  • Analyse de sentiment automatisée
  • Détection de cooccurrences (quels mots apparaissent ensemble?)
  • Visualisation des thèmes et de leurs relations
  • Classification automatique de segments de texte
  • Tout en gardant la possibilité de retourner au texte original

Revenons à R vs Stata

Maintenant que vous avez vu ce que R peut faire…

Avantages de R :

  • Plus de 21,000 packages spécialisés
  • Peut gérer plusieurs jeux de données simultanément
  • Meilleurs outils graphiques (ggplot2)
  • Connexion facile aux bases de données, APIs, web
  • Compétence très recherchée hors académie
  • Gratuit!

Avantages de Stata :

  • Fonctionnalités statistiques prêtes à l’emploi
  • Syntaxe plus intuitive pour les débutants
  • Documentation officielle très claire
  • Très répandu en économie et sciences sociales
  • Bon choix si temps limité pour apprendre

Alors, R ou Stata ?

Choisir selon vos besoins :

Choisir R si… Choisir Stata si…
Vous voulez une carrière hors académie Votre équipe utilise déjà Stata
Vous avez besoin de visualisations avancées Vous faites des analyses économétriques classiques
Vous travaillez avec des données variées (texte, web, etc.) Vous avez peu de temps pour apprendre
Vous voulez des analyses 100% reproductibles Vous préférez une interface plus guidée

Astuce

Notre recommandation : Apprenez R, mais gardez des notions de Stata pour la collaboration!

Installer R et RStudio

Quelle est la différence ?

  • R est le moteur, RStudio est le véhicule
  • R est le langage, RStudio est l’interface
  • RStudio facilite l’utilisation de R
  • C’est gratuit, les deux!

Les packages : étendre R

Les packages sont comme des applications pour R

  • R de base = fonctions essentielles
  • Packages = fonctions spécialisées créées par la communauté
  • Plus de 21,000 packages disponibles
  • Chaque package résout des problèmes spécifiques

Analogie

R de base = téléphone neuf Packages = applications qu’on télécharge selon nos besoins

Exemple : un graphique avec ggplot2

library(ggplot2)

# Données exemple
donnees <- data.frame(
  categorie = c("A", "B", "C", "D"),
  valeur = c(25, 40, 15, 20)
)

# Graphique
ggplot(donnees, aes(x = categorie, y = valeur, fill = categorie)) +
  geom_col() +
  labs(title = "Exemple de graphique avec R") +
  theme_minimal()
